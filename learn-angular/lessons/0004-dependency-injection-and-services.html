Dependency Injection & Services

You've built components and managed state with signals. Now let's learn how Angular's dependency injection (DI) system helps you share code and maintain separation of concerns.

Key Insight: Dependency Injection is Angular's way of providing dependencies (services) to components and other services. It promotes modularity, testability, and reusability.

What is Dependency Injection?

Instead of creating dependencies yourself, Angular provides them for you:

// ❌ Without DI - tight coupling
export class UserService {
  private http = new HttpClient(); // Creates its own dependency
}

// ✅ With DI - loose coupling
@Injectable({ providedIn: 'root' })
export class UserService {
  constructor(private http: HttpClient) {} // Angular provides HttpClient
}

Creating Services

Services are classes that provide shared functionality:

// Generate with Angular CLI: ng generate service services/user
import { Injectable } from '@angular/core';
import { HttpClient } from '@angular/common/http';
import { Observable } from 'rxjs';
import { User } from '../models/user.model';

@Injectable({ providedIn: 'root' })
export class UserService {
  private apiUrl = 'https://api.example.com/users';
  
  constructor(private http: HttpClient) {}
  
  getUsers(): Observable<User[]> {
    return this.http.get<User[]>(this.apiUrl);
  }
  
  getUser(id: number): Observable<User> {
    return this.http.get<User>(`${this.apiUrl}/${id}`);
  }
  
  createUser(user: User): Observable<User> {
    return this.http.post<User>(this.apiUrl, user);
  }
}

The @Injectable Decorator

@Injectable({ providedIn: 'root' })  // Singleton service available app-wide
export class LoggerService {
  log(message: string) {
    console.log(`[LOG] ${message}`);
  }
}
providedIn: 'root' makes the service a singleton available throughout the app. Angular tree-shakes unused services automatically.

Using Services in Components

@Component({
  selector: 'app-user-list',
  template: `
    @if (loading()) {
      <p>Loading users...</p>
    } @else {
      <ul>
        @for (user of users(); track user.id) {
          <li>{{ user.name }}</li>
        }
      </ul>
    }
  `
})
export class UserListComponent implements OnInit {
  users = signal<User[]>([]);
  loading = signal(true);
  
  // Inject service via constructor
  constructor(private userService: UserService) {}
  
  ngOnInit() {
    this.userService.getUsers().subscribe({
      next: (users) => {
        this.users.set(users);
        this.loading.set(false);
      },
      error: (err) => {
        console.error('Failed to load users:', err);
        this.loading.set(false);
      }
    });
  }
}
Modern Angular: With Angular 14+, you can use the inject() function instead of constructor injection: 
export class UserListComponent {
  private userService = inject(UserService);
  // ...
}

Injection Tokens

Sometimes you need to inject something that isn't a class (like configuration values):

// Define a token
import { InjectionToken } from '@angular/core';

export const API_URL = new InjectionToken<string>('API_URL');

// Provide the value
@NgModule({
  providers: [
    { provide: API_URL, useValue: 'https://api.example.com' }
  ]
})
export class AppModule {}

// Use the token
@Injectable()
export class ApiService {
  constructor(@Inject(API_URL) private apiUrl: string) {}
}

Service Hierarchies

Services can have different scopes:

Scope providedIn Behavior
Root 'root' Singleton app-wide
Component Component New instance per component
Module NgModule Singleton within module
Environment 'platform' Singleton across lazy modules 
// Component-scoped service (new instance per component)
@Injectable()
export class FormService {
  formData = signal({});
}

@Component({
  providers: [FormService] // Each component gets its own instance
})
export class FormComponent {
  constructor(private formService: FormService) {}
}
Think About It: When would you want a component-scoped service instead of a root singleton? What are the trade-offs?

Provider Configuration

// Different ways to provide services
@NgModule({
  providers: [
    // Basic
    UserService,
    
    // With value
    { provide: API_URL, useValue: 'https://api.example.com' },
    
    // With class
    { provide: LoggerService, useClass: ConsoleLoggerService },
    
    // With factory
    { 
      provide: UserService,
      useFactory: (http: HttpClient) => new UserService(http),
      deps: [HttpClient]
    },
    
    // Alias
    { provide: 'AliasedService', useExisting: UserService }
  ]
})
export class AppModule {}

Real-World Patterns

1. API Service Pattern

@Injectable({ providedIn: 'root' })
export class ApiService {
  private http = inject(HttpClient);
  
  get<T>(endpoint: string): Observable<T> {
    return this.http.get<T>(`${environment.apiUrl}/${endpoint}`);
  }
  
  post<T>(endpoint: string, data: any): Observable<T> {
    return this.http.post<T>(`${environment.apiUrl}/${endpoint}`, data);
  }
}

// Usage
@Injectable({ providedIn: 'root' })
export class UserService {
  private api = inject(ApiService);
  
  getUsers() {
    return this.api.get<User[]>('users');
  }
}

2. State Management Service

@Injectable({ providedIn: 'root' })
export class AppStateService {
  private userSubject = new BehaviorSubject<User | null>(null);
  user$ = this.userSubject.asObservable();
  
  // Signals version
  user = signal<User | null>(null);
  
  setUser(user: User) {
    this.userSubject.next(user);
    this.user.set(user);
  }
}

3. Authentication Service

@Injectable({ providedIn: 'root' })
export class AuthService {
  private router = inject(Router);
  private tokenKey = 'auth_token';
  
  isAuthenticated = signal(false);
  
  login(credentials: LoginRequest): Observable<AuthResponse> {
    return this.http.post<AuthResponse>('/auth/login', credentials)
      .pipe(
        tap(response => {
          localStorage.setItem(this.tokenKey, response.token);
          this.isAuthenticated.set(true);
          this.router.navigate(['/dashboard']);
        })
      );
  }
  
  logout() {
    localStorage.removeItem(this.tokenKey);
    this.isAuthenticated.set(false);
    this.router.navigate(['/login']);
  }
}

Testing with Dependency Injection

DI makes testing easier by allowing you to mock dependencies:

// Real service
@Injectable({ providedIn: 'root' })
export class UserService {
  getUsers() { return this.http.get('/users'); }
}

// Test with mock
describe('UserComponent', () => {
  let component: UserComponent;
  let mockUserService: jasmine.SpyObj<UserService>;
  
  beforeEach(() => {
    mockUserService = jasmine.createSpyObj('UserService', ['getUsers']);
    mockUserService.getUsers.and.returnValue(of([{ id: 1, name: 'Test' }]));
    
    TestBed.configureTestingModule({
      declarations: [UserComponent],
      providers: [
        { provide: UserService, useValue: mockUserService }
      ]
    });
    
    component = TestBed.inject(UserComponent);
  });
});
